데이터 & AI 세미나 핵심 정리

세미나 내용을 주제별로 묶어 정리

세미나 요약

데이터와 AI를 결합해 조직의 의사결정 방식을 바꾸는 것이 이번 세미나의 핵심 주제였습니다. 운영·분석 데이터를 통합한 플랫폼 위에서 자연어 기반 AI 에이전트가 데이터를 쉽게 찾아볼 수 있도록 구성하며, 대화형으로 대시보드를 구성해 누구나 데이터를 쉽게 활용할 수 있도록 제품에 대한 소개와 함께 제품에 AI 기능은 이제 필수적이라고 생각하게 되었습니다. 특히 Slack MCP로 권한을 부여하여 보안 관리 흐름을 만들어 에이전트들을 조정하는 end-to-end 거버넌스 체계를 구축한 부분이 흥미로웠으며, AI를 잘 활용하는 사람이 경쟁력을 갖는 시대에 맞춰, 데이터와 비즈니스를 연결하여 사용자가 원하는 시장의 가치를 찾고 그것을 목표로 적용해 상품성을 높여야 할 것이라고 생각하였습니다.

데이터 전략

데이터 자산화

데이터는 쌓일수록 자산이 된다. 운영 데이터와 분석 데이터가 함께 조화될 때 실질적인 가치가 생긴다.

  • 운영 데이터 + 분석 데이터 통합
  • 데이터를 넣고 사용성을 수집 → 자산으로 축적
  • 데이터 → 피처 → 모델로 진화
  • 사용자가 늘수록 데이터가 늘어나는 선순환 구조

목표 중심 데이터 활용

일반적 문제 해결이 아닌, 연결 대상과 목표를 먼저 정의하고 데이터를 활용해야 한다. 맥락을 알아야 구체적인 문제를 파악할 수 있다.

  • 목표와 대상을 먼저 정의, 데이터는 수단
  • 맥락 없이는 문제를 제대로 파악할 수 없다
  • 고객 가치와 연결된 데이터를 찾는다
  • 데이터는 문제를 발견하거나 해결하는 도구

데이터 플랫폼

플랫폼 구축 및 통합

수집·정제·배포를 하나로 연결하고, 산업별 니즈에 맞게 데이터를 제공하는 플랫폼이 필요하다.

  • 수집 → 정제 → 배포 파이프라인 통합
  • 산업별 니즈에 맞는 다양한 형태로 제공
  • 대규모 데이터로 비즈니스 선순환 구조 구축
  • 실행력을 강화할 수 있는 방법을 찾는다

비즈니스 시멘틱 정의

같은 데이터도 부서마다 다르게 해석한다. 예를 들어 매출·환불 데이터는 영업과 회계가 요구하는 기준이 다르다. 공통 정의가 필요하다.

  • 부서마다 다른 데이터 해석 → 공통 시멘틱 정의 필요
  • 사용자가 원하는 정보를 제공하는 테이블/플랫폼 구성
  • 어떤 데이터가 들어가느냐에 따라 결과 품질이 달라진다

AI 에이전트

에이전트 구축 및 자율 운영

자연어로 데이터와 대화하고, 에이전트들이 자율적으로 협업·학습하며 진화하는 구조를 목표로 한다.

  • 자연어 기반 데이터 에이전트 구축
  • 에이전트들이 자율적으로 운영·협업
  • 자가 학습 형태로 에이전트 진화 지원
  • 스냅샷 기능 — 즉시 롤백 가능

리서치 에이전트 (분석 자동화)

단순 조회를 넘어 "왜 매출이 올랐는가"처럼 원인을 탐색한다. 가설을 세우고 뒷받침 데이터를 찾아 리포트까지 자동 생성한다.

  • 단순 조회 → 원인 탐색 리서치 모드
  • 오픈 질문 기반 가설 수립 및 데이터 검증
  • 가설 → 검증 → 리포트 자동 생성
  • 에이전트와 나눈 분석 내용을 기록으로 저장

AI 대시보드

대화형 대시보드 설계

자연어 입력으로 시각화를 생성하고, 팔로업 질문이 가능한 대화형 인터페이스로 대시보드를 만든다.

  • 자연어 텍스트 → 분석 → 시각화 변환
  • 대시보드에는 의도가 있어야 한다
  • 팔로업 질문을 이어서 할 수 있는 구조

정확도 개선 및 할루시네이션 대응

AI가 잘못된 정의를 쓸 경우(예: 이탈 고객 정의) 사용자가 피드백을 주면 정확한 데이터로 재제공하는 구조가 필요하다.

  • 피드백 기반 데이터 정의 정교화
  • 할루시네이션 발생 시 사용자 피드백으로 보정
  • 비즈니스·데이터 맥락을 이해하는 에이전트 필요

에이전트 관리 및 거버넌스

에이전트 접근 관리

에이전트가 어떤 데이터에 접근하는지 파악하고 통제할 수 있어야 한다. 우리 데이터와 비즈니스 모델을 제대로 이해한 에이전트여야 한다.

  • 에이전트의 데이터 접근 경로 가시화
  • 권한 관리 — 어떤 정보를 가져오는지 제어
  • 비즈니스 맥락을 이해하는 에이전트 설계

성능 및 품질 모니터링

어떤 에이전트가 어떤 모델과 데이터를 사용하는지 추적하고, 품질을 지속적으로 관리하는 체계가 필요하다.

  • 에이전트별 모델·데이터 출처 추적
  • 성능 모니터링 및 품질 관리 체계 수립
  • 관리 체계 지속 개선

거버넌스 체계 구축

에이전트의 요청과 응답을 빠짐없이 기록하고, 보안을 유지하는 end-to-end 플랫폼으로 에이전트 전체를 조정·관리한다.

  • 보안 중심 거버넌스 체계 구축
  • 에이전트 요청·응답 트레이스 전체 기록
  • 에이전트를 조정하는 end-to-end 보안 플랫폼
  • Slack MCP로 권한 관리 — 동일한 권한 구성으로 보안 유지

멀티 모델 폴백 구조

여러 모델 중 하나가 응답하지 못하면 자동으로 다른 모델로 전환해 서비스 연속성을 유지하는 방식으로 AI가 작동한다.

  • 여러 모델을 선택적으로 사용하는 구조
  • 특정 모델 실패 시 다른 모델로 자동 전환 (폴백)
  • 안정적인 서비스 연속성 확보

조직 변화

AI 활용 조직으로의 전환

AI를 잘 쓰는 사람·조직이 경쟁력을 갖는 시대다. 데이터와 AI를 모든 구성원이 쓸 수 있게 만드는 것이 핵심 과제다.

  • AI를 잘 활용하는 사람이 경쟁력
  • 데이터와 AI를 모든 사람이 쓸 수 있어야 한다
  • AI 엔지니어가 AI를 다루는 시대
  • 대화형 인터페이스로 진입 장벽 낮추기
